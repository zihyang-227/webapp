Design Your Trial Flow

Trial Flow

Our trial flow design has two modes: simple and advanced.
Simple mode is easier to implement when you only need to have one stimuli on each page with three pages in total (e.g. IAT) and you don't need in-page logic.

Simple Trial Flow

Trial Timeline You can set the time you want each stimuli to show. For stimuli 3, it's typical the page awaiting for particiapnts to respond. You can set both Delay before keyboard and Delay after keybaord
Allowed Input Keys You can set the key inputs allowed for partcipants (e.g., in IAT, let K = "plesant", D="unpleasant")
Auto Proceed to next trail after timeout When you enable this button, it will not wait for paticipant's key responses and move to the next trial.

Advanced Trial Flow

Page Layout You can design how you want the page to look, it's especially effective for experiments that need two different types of stimuli to show at the same time. You can add stimuli shown on the page by clicing "+" on the right side and "+" on the bottom
Go to next page when Here we give three possibilities for researcher to choose
After fixed duration which is used for basic functioning of intermediate pages,
Keyboard response which is used for recording the keyboard responses participants make,
Mouse click response which is used for mouse tracking
Swap Display When you enable this button, it will randomly swap the positions of the stimuli.
Between Page Design

After click add page, you can set up both the interval between pages and the in-page logic (the little branch).
In-page Logic
This button is useful when you want to show this page depending on the results from the frist page's response.
For instance, in IAT, we want to give feedback for participants given their responses to different stimuli.
We can set we want this page to show when the response of Page# is/isnot correct keyboard response.
Moreover, we can also set this page to show when the selected pool of Page# is/isnot which pool.
We can also set the logic of these conditions by setting whether you want these conditions to be met the same time (AND) or only one needs to be met (OR).

Set Number of Trials and Rounds


After having designed one trial, it's time to decide how many trials you want in total and how many rounds you want.
Number of total trials Make sure the number of total trials don't excced any of your stimuli counts in one stimuli pool.
For instance, if you have four stimuli with 5 counts each in stimuli pool 1, your total countrs of trials can't exceed 20
Add more rounds When you want another copy of one round, just enable this button.
Notice that: when you embed this survey into qualtrics, you need to add another copy of our design block to have another round.
